## Codebase Patterns
- Models use SQLAlchemy `DeclarativeBase` with `Mapped` type annotations and UUID string PKs
- No Alembic migrations — new tables auto-created by `Base.metadata.create_all()`, new columns added via idempotent `ALTER TABLE` in `init_db()`
- All models must be imported in `init_db()` (database.py) AND exported from `models/__init__.py`
- Use `IF NOT EXISTS` pattern (try/except) for column migrations in `init_db()`
- JSON columns use `sqlalchemy.JSON` with `default=list` or `default=dict`
- Timestamps use `DateTime` with `default=lambda: datetime.now(timezone.utc)`
- Run `python -m py_compile` from `backend/` dir for quick typecheck; use `python -c "from app.models.X import Y"` to verify imports
- Git operations must be run from repo root `/mnt/c/Users/dobec/actions/`
- Backend Python runs from `/mnt/c/Users/dobec/actions/backend/`
- For LLM structured output via tool calling, use `chat_completion_with_tool(model, messages, tool_name, tool_schema)` in `llm_client.py` — handles both Anthropic tool_use and OpenAI function calling
- For planner-style calls with fallback chain, use `planner_completion(messages, tool_name, tool_schema, model_override=...)` — tries PLANNER_MODEL_CHAIN with API key checks
- For agentic tool calling loops (CodingAgent etc.), use provider SDKs directly; OpenAI and Anthropic have different tool call/result formats
- CodingAgent pattern: normalize messages to OpenAI dict format, convert for Anthropic at call time
- Pause/resume uses in-memory `pause_manager` singleton; CodingAgent checks between iterations; task status 'paused' is valid
- `diff2html` is available for rendering unified diffs; use `ColorSchemeType` enum from `diff2html/lib/types`, not string literals
- BaseAgent.stream_callback is set by executor before execute() — agents call `await self.stream_callback(chunk)` to publish LLM chunks via SSE
- Zustand store `taskStreamingText` holds per-task streaming text buffer; `appendTaskStreamingText(taskId, chunk)` appends
